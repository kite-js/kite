Kite is designed to write APIs more efficiently with latest Javascript features

Introduction

Kite is a web application framework, it's designed for efficient coding of APIs. Here are differences between other hot frameworks ( ExpressJS, KoaJS):

  • Kite is written in TypeScript with latest ECMAScript features: Decorators, Async function / await
  • Kite applications should be written in TypeScript too (recommended), it makes the source much simple, clear and maintainable
  • NO REST ROUTE in Kite (not yet), no route definition required in Kite, just place your API in a structured directory, the framework will map to your code, one api for one source file
  • Built-in input data filter, filters client inputs with rules like: "required/optional", "min/max value", "min/max length", "empty/nonempty string or array", "pattern" without losing performance (Kite generate filter codes for you)!
  • Built-in data type convertor, Kite converts input data to your declared types, it can convert input strings to JavaScript "Number", "Boolean", "Date", MongoDB "ObjectId" or any other types
  • MongoDB's losing document schema is here! By declaring a model class, Kite filters & converts input data before database operation, post JSON objects with nested objects / complex documents is pretty simple! And this feature is also works for SQL databases
  • Dependance injection supported, Kite manages dependent modules for you, each application service is created & managed in a single instance for reuse and data sharing
  • Enjoy coding, in development mode Kite detects file changes and reload relative modules within 2 seconds, so you don't need to stop & start your application server when source code is changed, this feature makes you focus on coding and testing. Unlike third-party node modules (nodemon / pm2) restart the whole application server, Kite only reloads the changed files and their relative modules.

Use Kite cli tools to quickly start your application!

                
npm i kite-tools -g
kite --help

Quick start

Installation

If you're going to start a Kite application manually, please make sure you've Node.js v7.10.0 or higher installed. And for development, TypeScript v2.4.0+ or higher is required, Visual Studio Code is recommended.

If everything is ready, then install Kite from npm:

npm i kite-framework

Configuration

Create a file "tsconfig.json" in your project root directory and copy & paste the following content into it, these configuration is for TypeScript compiler:

        
                
{
    "compilerOptions": {
        "moduleResolution": "node",
        "noImplicitAny": true,
        "target": "ESNext",
        "module": "commonjs",
        "sourceMap": true,
        "experimentalDecorators": true,
        "emitDecoratorMetadata": true,
        "declaration": false,
        "rootDir": "src",
        "outDir": "dist",
        "types": [
            "node"
        ],
        "typeRoots": ["node_modules/@types"]
    },
    "exclude": [
        "node_modules",
        "dist"
    ]
}

Application server

Create a folder named "src" for holding your ".ts" source files, then create a file "app.server.ts", like this:

        
import { Kite } from 'kite-framework';

Kite.init().fly();

Hello world

Create a folder "controllers" under "src" for holding APIs, then create your first API source file "greeting.ts": 

            
import { Controller, Entry } from 'kite-framework';
                    
@Controller()
export class GreetingController {
    @Entry()
    async exec() {
        return { message: 'Hello world!' };
    }
}

Compile & run

Run "tsc" command to compile the project:

        tsc

After this step, your project structure should looks like:

        
project_home/
├── src
│   ├── app.server.ts
│   └── controllers
│       └── greeting.ts
├── dist
│   ├── app.server.js
│   └── controllers
│       └── greeting.js
└── node_modules

Now run your first Kite application:

        node dist/app.server.js

Then open your browser and visit http://localhost:4000/greeting, you'll get this response:

        
{
    "message": "Hello world!"
}
