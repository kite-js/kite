Kite

Next generation framework for building APIs

Kite is designed to write APIs more efficiently with latest Javascript features

Current version 0.4.0:

  • Recompiled for new feature "async/await" with tsc option "--target ESNext"
  • More flexible controller metadata definition, enables middlewares to access these metadata
  • Removed "Context" class, request (IncomingMessage) & response (ServerResponse) now can directly get from entry point function parameter
  • Removed "Holder" & "HolderClass", privilege check is not a built-in function anymore, developer should implement this function in middleware
  • Removed controller metadata "method" check, developer should implement this function in middleware
  • Single Kite instance mode implemented, now can get Kite instance by "Kite.getInstance()" everywhere

Use Kite cli tools to quickly start your application!

                
npm i kite-tools -g
kite --help

Quick start

Installation

Kite requires Node.js v7.10.0 or higher to run. And for development, TypeScript v2.4.0+ or higher is required, Visual Studio Code is recommended.

If every is ready, install Kite from npm:

npm i kite-framework

Configuration

Create a file "tsconfig.json" in your project root directory and copy & paste the following content to it, this tells TypeScript compiler how to compile your project:

        
                
{
    "compilerOptions": {
        "moduleResolution": "node",
        "noImplicitAny": true,
        "target": "ESNext",
        "module": "commonjs",
        "sourceMap": true,
        "experimentalDecorators": true,
        "emitDecoratorMetadata": true,
        "declaration": false,
        "rootDir": "src",
        "outDir": "dist",
        "types": [
            "node"
        ],
        "typeRoots": ["node_modules/@types"]
    },
    "exclude": [
        "node_modules",
        "dist"
    ]
}

Application server

Create a folder named "src" for holding your ".ts" source files, then create a file "app.server.ts", like this:

        
import { Kite } from 'kite-framework';

Kite.init().fly();

Hello world

Create a folder "controllers" under "src" for holding APIs, then create your first API source file "greeting.ts": 

            
import { Controller, Entry } from 'kite-framework';
                    
@Controller()
export class GreetingController {
    @Entry()
    async exec() {
        return { message: 'Hello world!' };
    }
}

Compile & run

Run "tsc" command to compile the project:

        tsc

After this step, your project structure should looks like:

        
project_home/
├── src
│   ├── app.server.ts
│   └── controllers
│       └── greeting.ts
├── dist
│   ├── app.server.js
│   └── controllers
│       └── greeting.js
└── node_modules

Now run your first Kite application:

        node dist/app.server.js

Then open your browser and visit http://localhost:4000/greeting, you'll get this response:

        
{
    "message": "Hello world!"
}
